STACK / VORGABEN:
- Next.js mit TypeScript + Tailwind CSS.
- Prüfe zuerst, ob das Projekt App Router (Ordner `app/`) oder Pages Router
  (Ordner `pages/`) nutzt, und richte dich danach. Bei neuem Projekt: App Router.
- 3D mit React Three Fiber: @react-three/fiber, @react-three/drei, three.
- Smoothes Scrollen mit Lenis; Headline-/Scroll-Transforms optional mit framer-motion.
- Fonts via next/font/google: Saira Condensed, Barlow, Space Mono.
- Installiere benötigte Pakete:
  npm i three @react-three/fiber @react-three/drei lenis framer-motion
  npm i -D @types/three

NICHT VERHANDELBAR (sonst weicht das Ergebnis ab):
- Den HTML-Prototyp NICHT einbetten/iframen. Alles als saubere React-Komponenten
  + Tailwind neu aufbauen, exakt nach Doku.
- Die prozedurale Mars-Oberfläche aus reference/scene.js 1:1 übernehmen
  (alle Canvas-Helfer: makeNoiseCanvas, rgbaStr, softBlob, drawCrater,
  craterShape, genCraters, craterPath, drawDeepCrater, smoothPoles,
  buildMarsColor, buildMarsBump, starSprite). KEIN externes Mars-Foto verwenden.
  Die Krater-Liste (genCraters) zwischen Color- und Bump-Map TEILEN, sonst liegen
  Relief und Färbung nicht übereinander. smoothPoles MUSS aktiv bleiben
  (keine Naht/keine zusammenlaufenden Linien am Pol). Eiskappen (drawPolarCap)
  NICHT verwenden.
- Three/R3F nur client-side: Hero/MarsScene mit "use client"; das <Canvas> per
  next/dynamic mit { ssr: false } laden.
- Animationen STRIKT delta-time-basiert in useFrame((state, delta) => ...),
  niemals feste Schritte pro Frame (sonst ruckelt es).
- <Canvas dpr={[1, 1.5]}> (Pixel-Ratio cappen).
- Texturen nur EINMAL via useMemo erzeugen (4096x2048, teuer).
- Scroll-Fortschritt p (0..1) über einen useRef in die useFrame-Loop geben,
  NICHT pro Frame React-State setzen.

SCROLL-VERHALTEN (exakte Formeln, README §8.1):
  total = heroWrap.offsetHeight - window.innerHeight
  p     = clamp(-heroWrap.getBoundingClientRect().top / total, 0, 1)
  Headline:   translateX(-p*78vw),  opacity = max(0, 1 - p*1.5)
  ScrollHint: opacity = max(0, 1 - p*4)
  3D-Planet (Pivot, über easeInOut(p) interpoliert):
    x 1.28 -> 0.02,  y -0.04 -> 0.02,  scale 1.12 -> 1.72
    rotation.y = autoSpin(0.1 rad/s, delta-time) + p*PI*0.9
    Achsneigung rotation = [0.12, 0, 0.42] (Euler x,y,z)
  Kamera fov 45, position [0,0,3.25]. Lichter exakt aus PORTING_GUIDE §3.
  Sphere args [1,128,128], MeshStandardMaterial map+bumpMap, bumpScale 0.03,
  roughness 0.96, metalness 0. Sternenfeld 3200 Punkte (buildStars portieren),
  langsame Eigenrotation 0.011 rad/s.

STERNSCHNUPPE (README §8.3): 3px Punkt + Glow + 120px Schweif (CSS-Gradient),
  erster Flug nach 1500 ms, danach alle 9000 + rand*9000 ms. Start links-oben
  (sx W*0.24..0.58, sy H*0.05..0.29), Winkel 152..180°, Distanz W*0.4..0.66,
  Dauer 0.9..1.4s linear, opacity 0->1(8%)->1(78%)->0, z-index 0 (hinter Planet).
  Keyframes exakt aus reference/CADA Galaxy.html (.shooting-star, @keyframes shoot).
